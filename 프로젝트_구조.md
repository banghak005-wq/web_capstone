# 프로젝트 파일 구조

## 폴더 구조
```
backend/
├── main.py
├── api/
│   ├── __init__.py
│   ├── check_awaiting.py
│   ├── get_result.py
│   ├── join_awaiting.py
│   ├── login.py
│   ├── login_g.py
│   ├── logout.py
│   ├── me.py
│   └── result_ack.py
├── classes/
│   ├── __init__.py
│   ├── Job.py
│   └── Session.py
└── server_control/
    ├── __init__.py
    ├── check_session.py
    ├── create_session.py
    ├── remove_session.py
    └── response.py
```

---

## main.py
웹 프로세스의 진입점. 공유 자원 선언 및 라우팅 담당.
- `session_dict` : 세션 목록 (dict)
- `queue` : 대기열 (deque)
- `jobs_dict` : 작업 목록 (dict)

---

## api/
클라이언트(React)가 직접 호출하는 API 함수 목록. 하나의 파일에 하나의 함수만 있어야 함.

| 파일 | 매개변수 | 설명 |
|---|---|---|
| `login.py` | id, pw | db에서 id/pw 확인 후 세션 쿠키 발급 |
| `login_g.py` | - | 구글 로그인 (구조 미확정, 선언만) |
| `logout.py` | 세션 | 세션 목록에서 해당 세션 제거 |
| `me.py` | 세션 | session_dict에서 로그인된 유저 정보 반환 |
| `join_awaiting.py` | 세션, 프롬프트 | 이미지 생성 요청을 대기열에 등록 |
| `check_awaiting.py` | 세션 | 대기열 상태 확인 |
| `get_result.py` | 세션 | 완료된 이미지 결과 반환 |
| `result_ack.py` | 세션, 대기열id | 결과 수신 확인 후 해당 작업 메모리에서 삭제 |

---

## classes/
프로젝트에서 사용하는 데이터 클래스 목록.

| 파일 | 설명 |
|---|---|
| `Job.py` | 이미지 생성 작업 하나를 나타내는 클래스 |
| `Session.py` | 유저에게 발급되는 세션을 나타내는 클래스 |

---

## server_control/
api/ 폴더의 함수들이 내부적으로 사용하는 도우미 함수 목록.

| 파일 | 설명 |
|---|---|
| `check_session.py` | 모든 요청에 선행되며 세션 유효성 검사, 맵핑된 id 없으면 False 반환 |
| `create_session.py` | 세션 생성 후 session_dict에 추가 |
| `remove_session.py` | session_dict에서 세션 제거 |
| `response.py` | 모든 API 요청에 대응하는 통일형 JSON 반환 |
